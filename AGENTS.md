# VoiceBrain — AI Agent Context

## Overview

VoiceBrain is a personal AI voice assistant running as an n8n workflow. It processes Telegram messages (text, voice, photos) through a conversational AI Agent (GPT-4o-mini) with 9 integrated tools.

## Architecture

- **Runtime**: n8n workflow (23 nodes)
- **AI Model**: GPT-4o-mini (via n8n AI Agent node, typeVersion 3)
- **Input**: Telegram Bot API (webhook trigger)
- **Storage**: Memos (self-hosted note server)
- **Shopping**: Bring! API
- **Issues**: GitHub API (private repo support)
- **Search**: Brave Search API
- **Style Memory**: Pinecone Vector Store (1024 dims, text-embedding-3-small)

## Workflow Nodes (23 total)

### Input Path
1. **Message Trigger** — Telegram webhook (text, voice, photo, callback_query)
2. **Route Chat Input** — Switch node routes by message type
3. **Get audio file** → **Transcribe audio** — Voice note processing (Whisper)
4. **Photo to Text** — Extract caption from photos
5. **Parse Input** — Normalize all inputs, deduplication, chat history management
6. **Parse Callback** — Handle inline button responses
7. **Command Handler** — /start, /help, /tasks, /search

### AI Agent
8. **VoiceBrain Agent** — GPT-4o-mini AI Agent with system prompt and 9 tool connections
9. **Chat Model** — OpenAI Chat Model sub-node

### Tools (Code Tool nodes)
10. **Tool: Create Memo** — `specifyInputSchema: true` (title, body, tags)
11. **Tool: Add to Bring** — `specifyInputSchema: true` (items array)
12. **Tool: GitHub Issue** — `specifyInputSchema: true` (project, title, body)
13. **Tool: Search Memos** — `specifyInputSchema: false` (plain string query)
14. **Tool: Web Search** — `specifyInputSchema: false` (plain string query)
15. **Tool: Update Memo** — `specifyInputSchema: true` (memo_id, mode: append/replace/mark_done/replace_tag)
16. **Tool: Get Tasks** — `specifyInputSchema: false` (optional project filter)
17. **Tool: Send Buttons** — `specifyInputSchema: true` (question, options)
18. **Pinecone Vector Store** — retrieve-as-tool mode for style memory
19. **Embeddings OpenAI** — text-embedding-3-small (1024 dims)

### Output
20. **Format Reply** — Process agent output, store in chat history
21. **Send Reply** — Telegram sendMessage (HTML parse mode)
22. **Answer Callback** — Telegram answerCallbackQuery

## Conventions

### Tool Code Pattern
All toolCode nodes follow this pattern:
```javascript
try {
  const args = typeof query === 'string' ? JSON.parse(query) : query;
  // ... tool logic using args ...
  return `Success message`;
} catch(e) { return `Error: ${e.message}`; }
```

### Input Schema Rules
- Tools receiving **structured JSON** from the AI: `specifyInputSchema: true` + `jsonSchemaExample`
- Tools receiving a **plain string**: `specifyInputSchema: false`, use `query` directly
- Without proper schema, n8n rejects non-string inputs with "Expected string, received object"

### Environment Variables
All sensitive values are in n8n environment variables (not hardcoded):
- `MEMOS_URL`, `MEMOS_TOKEN`
- `GITHUB_PAT`
- `BRING_EMAIL`, `BRING_PASSWORD`
- `BRAVE_SEARCH_KEY`
- `OPENAI_API_KEY`

### Bring! API
Requires these headers on ALL requests:
```
X-BRING-API-KEY: ***REDACTED_BRING_CLIENT_KEY***
X-BRING-CLIENT: android
X-BRING-APPLICATION: bring
```

### n8n MCP Caveats
- `n8n_update_partial_workflow` replaces the entire `parameters` object — always include ALL fields
- `promptType` must be explicitly set to `"define"` (defaults to `"auto"` which breaks custom prompts)
- Workflow activation requires `webhookId` on trigger nodes (auto-generated by UI, must be manually set via MCP)

## Update Memo Modes

| Mode | Input | Behavior |
|------|-------|----------|
| `append` | `memo_id`, `new_content` | Reads memo, appends new content with separator |
| `replace` | `memo_id`, `new_content` | Replaces entire memo content |
| `mark_done` | `memo_id` | Replaces `#open` with `#done` in content |
| `replace_tag` | `memo_id`, `old_tag`, `new_tag` | Replaces one tag string with another |

All modes read the current memo first, then PATCH with updated content.
